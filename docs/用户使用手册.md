# Overlook 用户使用手册

> 一句话概括：以网格卡片快速浏览与操作多个网站的桌面看板。

## 简介

Overlook 是一个基于 Electron、React、TypeScript 和 Vite 构建的桌面应用。它提供可拖拽、可缩放的网页卡片网格，让你把常用站点放到一个看板上集中查看与操作，支持主题切换、自动刷新、布局导入导出和多平台打包发布。

项目主页：https://github.com/wavesbig/overlook

如果你觉得这个项目有用，请到 GitHub 为项目点 Star ⭐：https://github.com/wavesbig/overlook

## 安装与启动

- Windows：从 GitHub Releases 下载 `overlook-<version>-setup.exe`，双击安装后从开始菜单或桌面快捷方式启动。
- macOS：下载 `overlook-<version>.dmg`，打开后将应用拖入 `Applications`；首次运行如遇到“已阻止不受信任开发者”，在系统设置中允许运行。
- Linux：
  - AppImage：下载 `overlook-<version>.AppImage`，执行 `chmod +x` 后运行。
  - Debian/Ubuntu：下载 `overlook_<version>_amd64.deb` 并运行 `sudo apt install ./overlook_<version>_amd64.deb`。

> 提示：用户直接使用安装包，无需安装 Node.js；Node 仅在本地开发时需要。

## 系统要求

- Windows 10 或更高版本 / macOS 12+ / 常见 Linux 发行版（x64）
- 磁盘空间：建议 ≥ 300MB（含应用与缓存）

## 核心概念

- 卡片（WebView）：
  - 每张卡片绑定一个 URL，可选择 PC/移动端访问模式、刷新间隔、缩放比例、可选的目标选择器（用于加载后高亮页面元素）。
  - 每卡片使用独立持久化会话（`persist:gridcard-<id>`），Cookies/登录状态互不影响。
- 布局：
  - 一组卡片的位置与尺寸的集合，可创建多个布局并切换当前使用的布局。
- 设置：
  - 主题模式（浅色/深色/跟随系统）、字体大小；所有设置均本地持久化保存。

## 快速上手

### 添加卡片

1. 在仪表盘点击“添加卡片”，或将浏览器地址直接拖放到看板上自动创建卡片。
2. 填写：名称、URL；选择访问模式（PC/移动端）、刷新间隔（秒）、缩放、目标选择器（可选）。
3. 保存后，卡片会以默认大小（宽 8、高 20）添加到当前布局。

### 移动与调整大小

- 悬停卡片后，使用工具栏中的“拖拽柄”按钮进行拖动移动。
- 抓住卡片边缘拖动可调整尺寸；所有位置与尺寸均会自动保存。

### 切换布局

- 在侧边栏或顶部入口切换不同布局；支持重命名、删除与新建。

## 卡片操作

- 刷新：点击“刷新”，立即重新加载该卡片。
- 复制并添加：复制当前卡片为“副本”并添加到布局底部。
- 编辑：修改名称、URL、刷新间隔、访问模式、缩放等参数。
- 缩放：放大/缩小/重置 100%，每卡片的缩放会持久化保存。
- 全屏：将卡片置于全屏查看，再次点击恢复。
- 删除：移除卡片（支持删除确认）。
- 显示/隐藏控制：可隐藏工具栏与控件，专注查看内容。
- 自动刷新：当卡片处于可见且未最小化状态时，按设置的间隔自动刷新。

## 设置

- 主题模式：
  - 选择“浅色/深色/跟随系统”，应用会立即生效并记住设置。
  - 多数现代网站会根据 `prefers-color-scheme` 自动切换；不支持的站点可能需要手动切换或使用站点内设置。
- 字体大小：
  - 在设置页使用 90% ~ 150% 的预设调整整体界面文本的大小。
- 数据持久化：
  - 设置通过 Electron Store 存储在本地，重启后仍保留你的偏好。

## 布局管理

- 新建布局：输入布局名称后创建空白看板。
- 重命名/删除：对当前或其他布局进行管理；删除后若无布局会自动创建一个默认布局。
- 导入导出：
  - 右上角导出当前布局为 JSON。
  - 选择本地 JSON 即可导入并恢复布局与卡片配置。

## 数据保存与隐私

- 本地存储：
  - 使用 Electron Store 持久化保存：布局列表、当前布局 ID、全局设置（主题、字体）。
  - 每卡片会话独立，Cookies 与登录状态互不共享。
- 外部网络：
  - 应用不会上传你的数据；仅在你访问网站时按正常浏览器行为请求对应资源。

## 更新与发布

- 发布渠道：
  - GitHub Releases：提供 Windows（安装包）、macOS（DMG）、Linux（AppImage/DEB）。
  - 每次版本变更触发自动构建并直接发布正式版本（非草稿）。
- 自动更新：
  - 项目包含 `electron-updater` 依赖与示意配置文件 `dev-app-update.yml`。
  - 如需启用自动更新，需配置更新源并在主进程集成更新逻辑。

## 常见问题

- 页面打不开/白屏：
  - 检查 URL 是否正确；部分站点需要登录或具备特定权限。
  - 如是移动端页面，请切换“访问模式”为移动端再试。
- 主题不跟随：
  - 多数现代网站支持 `prefers-color-scheme`；若不跟随，可暂视为站点限制。
- 自动刷新不工作：
  - 仅当卡片可见时触发；请确保未最小化且卡片在视区之内。
- 导入失败或布局错乱：
  - 确认 JSON 结构与版本匹配；可先导出一个参考样例再修改。

## 快捷提示

- 快速添加：将浏览器地址直接拖放到看板即可生成卡片。
- 快速缩放：使用工具栏的“放大/缩小/恢复”三键。
- 精准定位：填写“目标选择器（CSS Selector）”可在页面加载后高亮目标元素。
